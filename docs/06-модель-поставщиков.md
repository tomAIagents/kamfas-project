# Модель работы с поставщиками

## Суть задачи

Клиент хочет купить минвату или арматуру → КамФас это не производит →
нужно показать товар, цену и организовать продажу автоматически.

## Типы товаров

| Тип | Пример | Источник цены | Автоматизация |
|-----|--------|--------------|---------------|
| Собственное производство | ЖБИ, тротуарная плитка | Свой склад | 98% |
| Прайс-файл поставщика | Кровля, утеплители | Excel/XML импорт | 70% |
| API поставщика | Металлопрокат | Реал-тайм запрос | 85% |
| По запросу (RFQ) | Нестандартные ЖБИ, тендеры | Котировка | 40% |

## Варианты интеграции с поставщиками

### Вариант А — Прайс-файлы (старт)
```
Поставщик → присылает Excel/XML (раз в день/неделю)
    ↓
Cron-задача импортирует + добавляет наценку %
    ↓
Клиент видит актуальную цену
    ↓
Заказ → уходит менеджеру → он заказывает у поставщика
```
**Плюсы:** просто, дёшево, быстро запустить
**Минусы:** цены не в реальном времени

### Вариант Б — API поставщика
```
Клиент открывает карточку товара
    ↓
Сайт делает запрос к API поставщика (за 0.3 сек)
    ↓
Получает: цена + наличие + срок поставки
    ↓
Добавляет наценку → показывает клиенту
    ↓
Заказ → автоматически создаётся у поставщика
```
**Плюсы:** реальные цены, полная автоматизация

### Вариант В — Кабинет поставщика
```
Поставщик → ЛК → вносит товары, цены, наличие
    ↓
Система добавляет наценку по правилу
    ↓
Заказ → уведомление в ЛК поставщика → он подтверждает
```
**Плюсы:** поставщики сами заинтересованы в актуальности

### Вариант Г — RFQ (запрос котировки)
```
Клиент выбирает материалы → нажимает «Получить КП»
    ↓
Система рассылает запрос 3–5 поставщикам
    ↓
Поставщики отвечают (авто или вручную за 1–2 часа)
    ↓
Система выбирает лучшую цену + считает доставку + наценку
    ↓
Клиент получает итоговое КП в PDF
```
**Плюсы:** идеально для крупных заказов B2B и тендеров

## Схема автоматического заказа (Вариант 2/3)

```
1. Клиент оформляет заказ
2. Система проверяет наличие (свой склад / прайс / API)
3. Если API → автоматически размещает заказ у поставщика
4. Рассчитывает доставку (СДЭК API / Деловые Линии / свой транспорт)
5. Формирует документы (СБИС)
6. Клиент получает: подтверждение + счёт + трек-номер
7. Автоуведомления на каждом этапе (WhatsApp + Telegram + Email)
8. Менеджер видит только исключения
```

## Расчёт доставки

| Тип груза | Способ | API |
|-----------|--------|-----|
| Стандартный | СДЭК | СДЭК API |
| Крупногабаритный | Деловые Линии | API |
| ЖБИ (плиты, блоки) | Манипулятор / кран | Свои тарифы по зонам |
| Локальная доставка | Собственный транспорт | Зоны в CMS |

## Панель управления наценкой (Admin)

```
Правила наценки:
  По категории:   Кровля +25%, Металлопрокат +15%, Крепёж +35%
  По поставщику:  Поставщик А +20%, Поставщик Б +18%
  По объёму:      от 100 000 ₽ → наценка -3%
  Для прорабов:   -5% к базовой наценке
  Минимальная маржа: не ниже 8% (защита от убытков)
```

## План запуска интеграций

| Этап | Срок | Действие |
|------|------|---------|
| Старт | Месяц 1–2 | Собственные товары + 2–3 поставщика (прайс-файлы) |
| Рост | Месяц 3–4 | Кабинет поставщика (10–15 партнёров) |
| Автоматизация | Месяц 5–6 | API металлопроката + RFQ-модуль |
| Маркетплейс | Год 2 | Поставщики конкурируют за заказ в реал-тайм |
